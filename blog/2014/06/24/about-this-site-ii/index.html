About This Site II

Jun 24th, 2014 | Comments

In order to make a more complete picture for the first post, I wanted to discuss a few of the (admittedly minor) customizations I’ve made to Octopress as well as my workflow for creating new content on the site. Hopefully this will help to make you a bit more confident when making your own Octopress site.

Disqus

Part of my hope for this blog was to provide an easy platform for discusson on my thoughts, but I wanted something that would be easy to manage, so since Octopress already contains built in support for Disqus it seemed like the natural choice. To get it fully functional, I had to set up an account on Disqus, put in my website info, and specify a Disqus short name. I then had to specify that Disqus short name in _config.yml as follows:

# Disqus Comments
disqus_short_name: sheindel
disqus_show_comment_count: true

And that’s it! I now have a comment section for my blog complete with a load of features from Disqus.

Word Cloud

One of the features I’ve always enjoyed about blogs is a Word Cloud. It provides a very concise and entertaining analysis of the content of ones blog. While I struggle with the level of flashiness I want to employ, I couldn’t help but try out a plugin that I found on the Octopress 3rd Party Plugins Page called Octopress cumulus. Following their simple instructions, I was able add it as an aside to my blog with little effort. If you create a blog, I highly recommend some type of visual tool like a Word Cloud to help visualize your categorization.

Google Analytics

This is probably the area that I am the least familiar with, but because I wanted to have some method of tracking my website traffic, I set up Google Analytics, which was incredibly easy, and found that unsurprisingly, I am the only visitor to my own website :–) In the same manner as Disqus, All I had to do was add my Google Analytics ID to the _config.yml file to make the changes take affect.

Blogging

This isn’t really a new feature, but I wanted to give you an idea of what is required to blog with Octopress and why I like it. So after the intial setup, here is what is required for me to publish a blog

Step 1: New Blog Post

So much command line goodness. All I have to do is browse to anywhere in my octopress folder to use the commands in the Rakefile. The primary command that I use from the standard Rakefile is new_post as follows

rake new_post["About This Site II"]
Step 2: Write…

The Rakefile will then inform me where the .markdown file wsa placed. At that point I use my favorite console based editor to write my technology ramblings.

Step 3: Build/Deploy

Once I have created some blogosphere junk, I only need to use the following command to build my website and push it up to my Github Pages repository where it will immediately be available to both of my readers on the world wide web.

rake gen_deploy

Alright, that’s enough talking about this website. Next time, I’ll cover some Python goodness. Maybe someone will even read them (once I finish my blogging seminar…)
